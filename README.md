# NIM - Game

## Introduction

NIM is a strategy game played by two players. Matches are arranged in several rows. The players take turns in taking the sticks away. A player must take one or more sticks. However, only matches from a single row may be taken.

The player who can take the last matchstick(s) wins.

## Exercise

A NIM game is to be developed as a console application. These requirements are to be taken into account:

### Requirements

- Initialisation of the game: The number of rows and number of sticks can either be entered by the user or randomly generated by the programme. Both variants are permissible.
- Display: The current score (number of sticks per row) must be displayed in an understandable form on the console. When the game is over, the winner should be displayed.
- Gameplay: The player takes turns with the computer. When it is the player's turn, a message prompts him to make his move.
- User turn: The user can enter in which row how many sticks are to be removed. Incorrect entries by the user should not cause the programme to crash.
- Computer move: The computer can make a valid move. A [winning strategy](https://www.hep-verlag.ch/pub/media/import/public/6486/mathematikaufgaben.pdf) can (but does not have to, but gives extra points) be implemented. Cf. different "levels" are also conceivable.
- Correctness: The moves must be applied correctly.
- Usability: With the help of the displayed messages, it should be clear to the user how to use the game.
